# TA-前台项目技术总览

## 1. 项目概况

教师工作台是一个面向教师日常教学、学员服务、任务处理和招生转化的综合型工作台。项目既包含课程管理、学员服务、任务处理等前台业务能力，也依赖权限、字段、工具、SOP 等后台配置能力，是一个典型的「业务复杂 + 配置驱动 + 多角色权限」系统。

项目的核心价值在于将教师日常分散的工作统一到一个平台内，通过课程视角、任务视角、学员视角和线索视角组织业务流程，并通过配置化能力降低新增字段、工具和模块的研发成本。

## 3. 技术栈经验

项目采用 Vue 3 + TypeScript + Vite 作为主技术栈，使用 Vue Router 管理路由，Pinia 管理全局状态。UI 层以 Ant Design Vue 为主，并结合 TailwindCSS、Less/Sass 和内部表格组件 `@xwfe/surely-vue-private-table`。

主要依赖能力包括：

- Axios：统一 HTTP 请求。
- ECharts：图表能力。
- dayjs：时间处理。
- lodash-es：通用工具函数。
- ali-oss：OSS 上传。
- @arms/js-sdk：ARMS 监控。
- @xwfe/web-track-browser：日志上报。
- vite-plugin-mock：本地 mock 数据。

工程侧使用 ESLint、Stylelint、Husky、Commitizen 保障代码质量和提交规范。需要特别注意内部 npm 包版本锁定，尤其是 `@xwfe/surely-vue-private-table`。

常用命令：

```bash
npm run dev
npm run build
npm run build:dev

```

## 4. 整体架构经验

项目的业务架构可按以下几个核心方向理解：

1. **课程与学员服务**：以「我的课程」为核心入口，承载课程卡片、课程详情、班级学员、讲次表现、班级日志、学情记录等能力。
2. **任务处理**：以「我的任务」为统一入口，聚合新学员触达、退费审核、批改、回放跟课、抖音退费回访等教师待办。
3. **工具体系**：分为批量工具和单体工具，分别解决多学员批量操作和单学员精细化操作。
4. **招生服务**：围绕私海、线索、成单明细、转化归因等场景，支撑销售线索管理与转化追踪。
5. **配置化基础设施**：通过字段服务、权限配置、SOP 配置、工具配置等机制，降低业务变更成本。

项目的关键架构特征是「本地路由 + 后端权限配置 + 字段服务 + 工具注册」共同驱动页面能力。新增页面、模块、字段和工具时，通常需要同时关注前端代码、后端权限配置和配置后台。

## 5. 我的课程模块经验

「我的课程」是教师工作台的核心模块，主要由三部分组成：

- 多课程卡片列表。
- 单课程信息汇总。
- 教师工作台表格服务。

目录重点在 `src/pages/newMyClass/`：

```text
src/pages/newMyClass/
├── index.tsx
├── api.ts
├── type.ts
├── baseData.ts
└── components/
    ├── SearchDemand/
    ├── CardList/
    ├── DataEdit/
    ├── CourseDataList/
    └── LessonDataList/
```

课程卡片列表由于后端需要从多个数据源获取课程信息，无法分页加载，因此前端需要一次性拉取全部数据，再在前端完成筛选。

课程详情页以 `ClassList` 为容器，由顶部课程切换器、左侧模块列表和模块内容区组成。模块包括：

- 班级学员。
- 讲次表现。
- 全讲次表现。
- 班级日志。
- 学情记录。

课程卡片页和课程详情页实际都在 `src/pages/newMyClass/index.tsx` 下作为子路由渲染。切换模块时，本质是切换子路由，并通过 query 携带 `selectedCourseId`、`selectedClassId`、`mod_id`、`selectModule`、`system_sop_ids`、`pageSource` 等关键参数。

该模块的经验是：课程、班级、模块三者状态高度耦合，切换课程或长短期班时，要特别注意当前模块是否还存在。如果找不到原模块，需要回退到模块列表第一个模块，避免页面状态失效。

## 6. TWS 表格服务经验

TWS 表格的设计目标是减少新增字段时的前后端重复开发成本，通过字段化配置实现自动接入。

核心数据流：

```text
页面初始化
  ↓
getColumnsInfo()
  ↓
并行请求 getHeadOrder() 和 getAppSetting()
  ↓
getHeadHiddenData 合并字段配置与用户设置
  ↓
TableDeal 数据格式化
  ↓
渲染到 Table 组件
```

字段配置中需要重点关注：

- `field_id`：字段 ID。
- `field_name`：字段 key。
- `field_title` / `frontend_field_title`：后端字段标题和前端展示标题。
- `frontend_field_show_genre`：单元格显示类型。
- `frontend_header_field_show_genre`：表头显示类型。
- `frontend_search_rule`：搜索规则。
- `frontend_filter_rule`：筛选规则。
- `frontend_sort_rule`：排序规则。
- `frontend_interact_rule`：交互规则。
- `reflections` / `enum`：枚举映射。
- `hidden` / `newHidden`：默认隐藏与用户自定义隐藏。
- `fixed` / `width` / `sort`：列固定、宽度和排序。

渲染机制分三层：

1. `specialRow`：处理特殊字段。
2. `operateRow`：处理操作类字段。
3. `showRow`：处理普通展示字段。

筛选和排序同时支持前端与后端两种模式。前端筛选通过 `filterFn()` 处理，后端筛选通过 `serverExpose()` 发送条件。排序则根据 `frontend_sort_rule` 选择排序算法，也可发送排序参数到后端。

表格能力的关键经验是：字段展示不是写死的，而是由字段服务配置驱动。新增字段时优先判断字段平台能否覆盖展示、筛选、搜索、排序、导出等能力，只有配置无法满足时再补充前端渲染或转换逻辑。

## 7. 批量工具经验

批量工具用于帮助教师批量处理学员或班级相关工作，核心目录在 `src/components/batchTools/`。工具类型覆盖消息群发、学员管理、班级管理、榜单生成、企微备注、学习资源发送、字段同步等场景。

批量工具的核心经验是：展示权限来自后端配置，但执行能力必须前端显式注册。也就是说，权限包中配置了工具后，前端还需要在对应工具映射中注册执行逻辑，否则会出现「工具未接入」。

典型工具包括：

- 短信群发：支持模板、变量替换、风险过滤、预览和定时发送。
- 企业微信群发：支持文字、图片、文件、视频、学习报告、阶段报告、错题本、课程资料等多种消息类型。
- 批量标注：支持输入型字段、选择型字段、多选字段和自定义标签。
- 释放学员：适用于私海学员资源释放。
- 调整小班：支持批量迁移学员到目标班级。
- 班级群绑定与创建：管理企业微信群与班级关系。
- 生成榜单：支持讲次榜单、课程榜单、自定义榜单、万能榜单。
- 阶段报告、口算小程序、课下考试、诊断选择等学习资源类工具。

批量工具设计中需要重点关注风险过滤和操作前校验，例如免打扰、时段限制、退费学员过滤、是否已选学员、双端上课校验、企微在线状态等。

## 8. 单体工具经验

单体工具面向单个学员的精细化操作，目录在 `src/components/singleTools/`。它与批量工具能力有交集，但操作粒度更细，适合在表格行级操作中使用。

典型工具包括：

- 添加企微好友。
- 外呼工具。
- 互动题查看。
- 错题重做。
- 学习报告、阶段报告、结课报告、学习日记。
- 诊断工具。
- 标记完成。
- 去调课。
- 上传图片。
- 录音检查。
- 伴学加入链接。
- 代购工具。
- 销售工具。
- 企微备注工具。

单体工具的关键经验是：多数工具都依赖当前学员、课程、班级、讲次等上下文参数，因此接入时要明确工具所需参数来源，并保证表格行数据、路由 query、模块配置之间能正确传递。

## 9. 我的任务模块经验

「我的任务」是教师日常工作的核心入口，用于统一聚合待办事项。它通过任务卡片展示待处理数量、即将超时、超时等关键状态，帮助教师快速处理高优任务。

主要任务包括：

- 新学员触达。
- 退费审核。
- 批改。
- 回放跟课。
- 抖音退费电话回访。

任务首页默认路径为 `/myTask/task`，通过 `/haichuan/api/task/home` 获取首页数据。任务卡片包括图标、名称、简介、数据统计和更新时间提示。长时间停留时需要提醒用户刷新，避免任务数据过期。

### 9.1 新学员触达

新学员进入班级后，教师需要在规定时间内完成首次触达。该模块路径为 `/myTask/stuTouch`，支持复制姓名、群发短信、完成触达、电话、发送加微申请等能力。

关键业务规则是：主动沟通后必须点击「完成触达」才会真正完成任务，单纯拨打电话或发送消息不会让任务从列表消失。

### 9.2 退费审核

退费审核路径为 `/myTask/refundReview`，教师需要在规定时间内确认是否同意退费。操作流程通常是电话沟通、上传截图、确认或取消退费。该模块任务时效短，建议每天早晚检查。

### 9.3 批改

批改路径为 `/myTask/correctOverview`，用于处理作业或考试批改。需要关注主体差异：部分字段仅在希望学主体展示。接口字段中 `homework_source` 和 `homework_type` 需要映射为前端字段 `homework_source_id` 和 `homework_type_id`。

### 9.4 回放跟课

回放跟课路径为 `/myTask/replayCourse`，用于跟踪学员观看回放课程情况。支持复制姓名、导出表格、群发企微消息、去跟课等能力。部分批量工具通过主体差异配置控制。

### 9.5 抖音退费电话回访

抖音退费电话回访路径为 `/myTask/returnVisit`，仅在优课主体展示。完成条件包括电话打通并填写必填字段，或两次电话未打通但填写必填字段。主动完成工作后仍必须点击「完成回访」。

## 10. 公共组件经验

任务详情页大量复用公共组件，主要包括：

### 10.1 OperationTable

`OperationTable` 提供统一操作工具栏，包含：

- 表格字段配置。
- 刷新数据。
- 格式化表格。
- 批量工具栏。
- 统计信息展示。

常见参数包括 `origin`、`column`、`defaultColumn`、`onFormatFn`、`onSetColumn`、`onRefresh`、`batchList`、`selectedRows`、`checkNum`、`isFilter`、`total`、`optionParams` 等。

### 10.2 Table

项目自定义 Table 基于 Ant Design Vue Table 封装，支持列配置、筛选、搜索、排序、行选择、单体工具、学生详情联动等能力。

### 10.3 StudentDetail

学生详情抽屉用于展示学生基本信息、标签、学习数据、沟通记录和快捷操作。通常点击学生姓名时打开，并通过学生列表支持左右切换。

### 10.4 面包屑与表格配置

任务详情页使用 `useBreadcrumb` 设置面包屑。表格列配置通过 `getColumnsInfo` 获取并持久化，支持显示/隐藏、排序等用户自定义能力。

公共组件的核心经验是：任务页、课程页和招生页都在复用同一套表格和工具能力，开发新模块时应优先复用 `OperationTable`、`Table`、`StudentDetail` 和字段服务，避免重复实现。

## 11. 招生服务模块经验

招生服务围绕私海系统展开，用于管理不同角色的销售线索，支撑线索分配、跟进、成单转化和渠道归因。

私海类型包括：

- 我的私海：普通销售或班主任使用，支持线索数、今日已跟进、今日新分配、自动释放、优惠券查看、批量操作和单行工具。
- CC 私海：课程顾问使用，重点关注入班转化，支持按教师筛选。
- 班主任私海：班主任使用，支持小红书线索、老带新线索、常规线索等多 Tab 和学科维度展示。
- 我的私海（高）：高价值线索处理人员使用，区分 leads / other 等来源，支持自动释放。

成单明细用于追踪学生信息、订单信息、成单信息、转化归因和退费信息。小红书成单明细专门追踪小红书渠道，并支持首单标识。

招生服务核心业务指标包括：

- 线索数。
- 今日已跟进。
- 今日新分配。
- 入班学员数。
- 成单数。
- 成单金额。
- 退费率。
- 关联任务数。

线索流转流程是：线索进入系统 → 自动/手动分配 → 进入对应私海 → 销售跟进 → 成单转化 → 记录成单明细。

该模块的关键经验是：不同私海类型共用基础组件，通过配置区分展示模块；同时需要处理数据权限，不同角色只能查看其权限范围内的线索。

## 12. 权限与配置化经验

项目中权限和配置贯穿菜单、路由、模块、字段、工具。

关键经验：

- 页面是否可访问，通常由本地路由和后端权限配置共同决定。
- 菜单是否展示，取决于后端权限接口与前端菜单配置的匹配。
- 模块是否展示，依赖权限树中的模块配置和 `menu_info`。
- 字段是否展示，依赖字段服务、用户自定义列配置和表格渲染规则。
- 批量/单体工具是否展示，依赖 `menu_info` 中的工具配置。
- 工具是否可执行，依赖前端是否完成 action 映射和 callback 注册。

因此新增能力时不能只改前端代码，需要同步检查：

1. 路由是否配置。
2. 后端 URL 权限是否配置。
3. 角色权限包是否关联。
4. 菜单或模块配置是否返回。
5. 字段服务或工具配置是否生效。
6. 前端渲染、映射、执行逻辑是否注册。

## 13. 日志与监控经验

项目接入了 ARMS 和业务日志。关键页面与工具操作需要通过 `sendLog` 上报。

典型场景包括：

- 点击任务卡片。
- 页面截图。
- 工具使用。
- 课程、班级、模块切换。

日志中通常需要传递页面名称、页面路径、品牌主体、业务来源、课程 ID 等信息。工具使用日志常通过 `_IdentifierOrigin` 传递来源标识。

经验上，新接入页面或工具时需要同步考虑埋点，否则后续无法定位使用情况和业务转化效果。

## 14. 品牌与主体差异经验

项目中存在希望学、素养、优课等主体差异。差异可能体现在：

- 菜单是否展示。
- 页面是否启用。
- 字段是否展示。
- 工具是否可用。
- 任务是否存在。
- 文案和背景图是否不同。

例如抖音退费电话回访仅在优课主体展示；批改模块部分字段仅在希望学主体展示；回放跟课部分工具通过主体配置控制。

处理主体差异时应优先使用已有品牌配置或差异配置，避免在业务代码中散落硬编码判断。

## 15. 常见问题经验

- 完成实际沟通或操作后，任务不一定自动消失。部分任务必须点击「完成触达」「完成回访」等完成按钮。
- 长时间停留任务页面可能导致数据过期，需要刷新或重新进入页面。
- 表格筛选重置通常通过格式化按钮触发，部分场景通过销毁重建组件重置筛选状态。
- 导出能力不是所有模块都有，通常由批量工具配置决定。
- 接口字段和前端字段可能不一致，需要在数据处理层做字段映射。
- 表格行需要稳定 `rowKey`，部分列表会补充 `origin_index`。
- 时间戳字段需要统一格式化，例如用 `dayjs.unix(...).format('YYYY-MM-DD HH:mm:ss')`。

## 16. 工程落地建议

### 16.1 新增页面

新增页面时应先确认是否需要权限配置。如果需要权限，需要同时处理路由、菜单、后端 URL 配置、角色权限包和前端页面。

### 16.2 新增字段

新增字段应优先走字段服务。先确认字段展示、搜索、筛选、排序、导出规则是否可配置；若字段服务无法覆盖，再补充前端 `HeaderCom`、`RowCom`、`specialRow`、`operateMap`、`showRowMap`、导出转换等逻辑。

### 16.3 新增批量工具

新增批量工具时，需要先在配置后台创建工具并挂到权限包模块下，再在前端按 action 注册工具执行逻辑。还要补充选人校验、风险过滤、主体差异和操作反馈。

### 16.4 新增任务

新增任务应复用任务首页卡片、任务详情页表格、`OperationTable`、批量工具、单体工具和面包屑能力。注意任务统计口径、超时规则、完成规则和刷新提示。

### 16.5 新增招生服务能力

新增私海或成单能力时，应先确认角色、数据权限、线索来源、转化归因、筛选维度和批量/单行工具。尽量复用现有私海基础组件，通过配置区分差异。
